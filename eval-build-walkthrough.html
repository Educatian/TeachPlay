퀄트릭스 서베이를 클코에 시켜서 만들기

Claude Code (또는 Codex CLI)에 그대로 붙여넣을 8개 프롬프트로 토큰 연결부터 응답 수집까지. 코드 한 줄 안 쓰고 따라할 수 있게 정리했고, 마지막에 한 케이스 (TeachPlay 사전·사후 설문)를 붙였습니다.

준비물: 퀄트릭스 API 토큰 한 줄 + 데이터센터 ID  ·  도구: Claude Code 또는 Codex CLI  ·  걸리는 시간: 처음 한번 30~40분, 이후 5분이면 신규 설문 한 건

Part 1

따라하기 8단계

각 단계마다 당신이 넣을 프롬프트(빨간 박스), 클코가 답하는 모양(파란 박스), 결과로 보이는 것(초록 박스)을 함께 보여줍니다. 위에서 아래로 그대로 따라가면 됩니다.

Step 01 토큰 연결 처음 한 번만

본인 퀄트릭스 계정에서 API 토큰을 받아와 텍스트 파일로 저장한 뒤, 클코에게 그 파일 경로만 알려주면 됩니다. 토큰 자체를 채팅창에 붙이지 마세요.

토큰 받는 법. 퀄트릭스 로그인 → Account Settings → Qualtrics IDs → API 섹션에서 Token(40자)과 Datacenter ID(예: pdx1) 확인 → 본인 컴퓨터 어딘가의 .txt 파일에 저장.
P당신 → 클코
내 퀄트릭스 토큰 파일이 [여기 경로 적기]에 있어. 그 토큰으로 내 계정에 잘 연결되는지 확인해줘. 계속 쓸 거니까 다음에 또 안 부르도록 메모리에 저장해두고.
C클코 → 당신

파일을 읽어서 토큰 한 줄을 추출하고, /whoami 엔드포인트로 검증할게요.

  • 토큰 파일 읽음 → 40자 토큰 추출
  • 퀄트릭스에 호출: GET /API/v3/whoami
  • 응답: 브랜드 / 사용자명 / 데이터센터 확인
  • 다음 세션에서 다시 묻지 않게 위치 기록 완료
✓이렇게 보일 거예요
  • "연결 성공 — 브랜드 universityofalabama, 사용자 jmoon19#…, 데이터센터 pdx1" 같은 한 줄 보고
  • 다음 단계부터는 토큰 다시 안 물음

Step 02 빈 서베이 만들기 셸 하나

먼저 빈 껍데기 서베이부터 하나 만듭니다. 이름만 정해주면 됩니다. 중요한 건 비활성 상태로 만든다는 점 — 응답이 들어오지 않게.

P당신 → 클코
"우리 프로젝트 사전 설문"이라는 빈 서베이 하나 만들어줘. 영어로, 비활성(isActive=false) 상태로. 서베이 ID는 알려주고, 다른 작업은 아직 하지 마.
C클코 → 당신

POST /survey-definitions 한 번 호출했어요.

  • 서베이 이름: "우리 프로젝트 사전 설문"
  • 언어: EN · 카테고리: CORE
  • 비활성 상태로 자동 생성됨
✓이렇게 보일 거예요
  • SurveyID: SV_xxxxxxxxxxxxxxx 한 줄
  • 퀄트릭스 웹 UI → Projects 탭에 새 서베이가 즉시 보임 (빈 상태)
  • 다음 단계에서 이 ID를 계속 씀

Step 03 문항 넣기 자연어로

여기서부터가 핵심입니다. 5점 척도, 단답, 매트릭스 같은 척도 종류를 직접 지정 안 해도 됩니다. 그냥 측정하고 싶은 걸 자연어로 말하세요. 클코가 표준 양식에 맞춰 페이로드를 짜서 넣어줍니다.

P당신 → 클코
방금 만든 서베이에 사용성(usability) 문항 4개를 5점 동의-Likert로 넣어줘. - 이 시스템은 내 요구를 충족한다 - 이 시스템은 사용하기 쉽다 - 이 시스템을 쓰는 게 답답하다 (역코딩) - 잘못된 걸 고치는 데 시간을 너무 많이 썼다 (역코딩) 다 필수응답으로, 분석용 태그는 USAB_1 ~ USAB_4 로 부탁.
C클코 → 당신

4개 문항을 단일선택 수직 라디오(5점, Strongly disagree → Strongly agree)로 만들어 추가했어요.

  • 문항 4개 × POST /questions
  • DataExportTag: USAB_1, USAB_2, USAB_3, USAB_4 (CSV 컬럼명으로 쓰임)
  • ForceResponse: ON
  • 아직은 기본 블록에 다 들어가 있음 — 다음 단계에서 블록 나눌 수 있음
✓이렇게 보일 거예요
  • 퀄트릭스 UI에서 서베이 열면 4개 문항이 5점 척도로 보임
  • Preview 버튼으로 응답자 시점 미리보기 가능
다른 척도가 필요하면. "단답 텍스트로", "여러 항목 매트릭스로 (행이 항목, 열이 5점)", "체크박스로 여러 개 선택", "0~10 슬라이더로" 같이 자연어로 말하면 됩니다. 매트릭스나 슬라이더는 양식이 까다로워서 클코가 한 번 더 확인 질문을 할 수 있어요.

Step 04 블록 나누기 섹션 구분

문항이 늘어나면 섹션별로 나누는 게 분석에도 응답자 경험에도 좋습니다. 클코에게 섹션을 어떻게 나눌지 자연어로 알려주면 됩니다.

P당신 → 클코
방금 넣은 4개는 "사용성" 블록으로 묶어줘. 그리고 새 블록 두 개 더 만들어: 1) "AI 사용 자신감" — 5점 자신감-Likert 문항 4개 (예: 챗GPT/클로드를 수업용 자료 초안에 쓸 수 있다 등 — 좋은 4개로 알아서 짜줘) 2) "교사 효능감 전이" — 5점 자신감-Likert 문항 3개 (내가 만든 자료로 학생들이 잘 배울 거다 같은 톤) 태그는 AISE_1~4, TSE_1~3 으로.
C클코 → 당신

블록 3개 정리, 문항 7개 추가했어요.

  • 기본 블록 → "1_사용성" 으로 이름 변경 (USAB_1~4)
  • 새 블록 "2_AI 사용 자신감" 생성 + AISE_1~4 4개 (Bandura 식 task-specific 4문항)
  • 새 블록 "3_교사 효능감 전이" 생성 + TSE_1~3 3개
  • 총 11문항. 응답 시간 추정 ~3분
✓이렇게 보일 거예요
  • UI에 좌측 사이드바로 블록 3개가 트리 구조로 보임
  • 각 블록을 펼치면 그 블록의 문항만 보임
  • Preview에서는 블록 간 자동 페이지 분리됨

Step 05 코호트 태그 URL 한 줄로 그룹 구분

한 서베이를 여러 그룹(예: 봄 코호트 / 여름 코호트, 학교 A / 학교 B)에 보낼 때, URL 끝에 ?cohort=… 만 다르게 붙여 보내면 응답 데이터에 자동으로 그 태그가 따라 들어옵니다.

P당신 → 클코
이 서베이에 cohort 라는 URL 파라미터 받게 해줘. 배포할 때 ?cohort=2026-spring 같이 붙여 보내면 응답에 그대로 기록되도록.
C클코 → 당신

서베이 플로우 첫머리에 EmbeddedData 선언을 추가했어요.

  • URL의 ?cohort=2026-spring 값이 응답 CSV의 cohort 컬럼으로 떨어짐
  • 참가자에게는 안 보이는 메타 데이터
  • 여러 그룹별로 다른 링크를 보내고 싶으면 같은 서베이에 다른 cohort 값만 붙이면 됨
✓이렇게 보일 거예요
  • UI → Survey Flow 탭 첫 줄에 "Set Embedded Data: cohort" 보임
  • 나중에 응답 export 하면 cohort 컬럼이 다른 응답 컬럼과 함께 들어옴

Step 06 브랜드 테마 기관 색·폰트로

기본 퀄트릭스는 무난한 회색-파랑입니다. 본인 기관 색·로고로 바꾸면 응답률이 의미 있게 올라갑니다(인지된 정당성 ↑). 클코에게 색 코드와 헤더 문구만 주면 됩니다.

P당신 → 클코
이 서베이에 우리 학교 브랜드로 테마 입혀줘. - 메인 색: #9E1B32 (UA Crimson) - 폰트: Inter - 헤더: 좌측 Crimson 4px 보더 + "MY PROJECT — University of Alabama, College of Education" - 푸터: "익명 연구 도구 · 개인정보 미수집 · UA IRB 승인" 한 줄 - 진행률 표시(progress bar) 켜고, 뒤로가기 버튼 활성화 다른 옵션값은 건드리지 말고.
C클코 → 당신

기존 옵션을 가져와서(GET) 위 항목만 덮어쓰고 다시 PUT 했어요(idempotent).

  • 버튼 / progress bar / hover 색 → Crimson
  • 본문 폰트 → Inter (시스템 폴백)
  • 헤더 / 푸터 HTML 적용
  • SurveyProtection, PartialData 같은 필수 키는 그대로 보존
✓이렇게 보일 거예요
  • Preview 누르면 위 오른쪽 모크업처럼 Crimson 톤으로 렌더
  • 모바일 응답도 같은 테마 유지 (반응형)
  • 브라우저 탭 제목도 "MY PROJECT — Pre/Post Survey" 식으로 바뀜

Step 07 활성화 · 배포 응답 받을 준비

여기까지는 모두 비활성 상태였습니다. IRB 검토 / 자체 점검이 끝났을 때만 활성화하세요. 활성화는 명시적 한 줄로 분리되어 있습니다 — 실수로 응답 받기 시작하는 일이 없게.

P당신 → 클코
서베이 활성화하고, 익명 배포 링크 하나 발급해줘. 만료일은 2026-12-31 자정. 링크 끝에 ?cohort= 어떻게 붙이면 되는지도 같이 알려주고.
C클코 → 당신

두 단계로 처리:

  • PUT /surveys/{id} → isActive: true
  • POST /distributions → 익명 링크 + 만료일
✓이렇게 보일 거예요
  • 익명 링크 한 줄: https://<dc>.qualtrics.com/jfe/form/SV_xxx
  • 코호트별 발송 시: 위 링크 끝에 ?cohort=2026-spring(또는 본인 태그) 붙여서 보내기
  • 응답이 들어오기 시작하면 UI → Data & Analysis 탭에서 실시간 카운트

Step 08 응답 export 분석용 CSV

응답이 어느 정도 쌓였을 때, 자기 컴퓨터로 CSV 파일 받아오는 단계. 클코가 비동기 작업(요청 → 폴링 → 다운로드)을 알아서 처리합니다.

P당신 → 클코
지금까지 들어온 응답 다 CSV로 받아서 [원하는 폴더 경로]에 풀어줘. 컬럼명은 내가 정한 분석 태그(USAB_1 등) 그대로 유지되도록.
C클코 → 당신

비동기 export 작업 시작 → 완료 폴링 → zip 다운로드 → 풀어 저장.

  • POST /export-responses → progressId 받음
  • 몇 초마다 status 확인하다 complete 되면 fileId 받음
  • GET /export-responses/{fileId}/file → zip → 풀기
  • CSV에 cohort, USAB_1~4, AISE_1~4, TSE_1~3 컬럼 모두 들어감
✓이렇게 보일 거예요
  • 지정한 폴더에 SurveyName_responses.csv 한 파일
  • 한 행 = 한 응답자, 컬럼명 = 분석 태그
  • 이제 R / Python / SPSS 어느 쪽으로든 분석 시작
여기까지가 한 사이클. 다음 코호트가 들어오면 Step 5의 cohort 값만 바꿔서 링크 새로 발급(Step 7) → 응답 누적 → Step 8 다시. 신규 설문이 필요하면 Step 2부터 다시 시작하면 되고, 토큰(Step 1)은 다시 건드릴 필요 없습니다.

Part 2 · 케이스

TeachPlay 사전·사후 설문

위 8단계를 실제 프로젝트에 한 번 적용한 사례. UA의 12-세션 AI 게임 디자인 마이크로크레덴셜 평가용으로 사전 20문항 + 사후 22문항을 한 번의 대화로 만들었습니다.

Case 01 한 번에 시키기 통합 프롬프트

측정 의도와 제약을 한 메시지에 다 적어주면 클코가 Step 2~6까지 한 번에 처리합니다. 설계가 끝나있을 때 유용한 압축 형태입니다.

P당신 → 클코
TeachPlay 프로젝트(AI 게임 디자인 마이크로크레덴셜) 평가용으로 사전·사후 설문 두 개를 한꺼번에 만들어줘. [사후 22문항] 비활성, perception × efficacy 두 축 - Tag 3개: 매칭코드 / 역할 / 실제 투입 시간(버킷) - 사용성 4 (UMUX, 5점 동의) - AI 사용 자신감 4 (5점 자신감, 사전과 동일 문구) - 자격증명 가치 인식 4 (signal/social/economic/realized 단일 합성) - 디자이너 정체성 4 (사전과 동일 문구) - 교사 효능감 전이 3 [사전 20문항] 비활성 - 매칭코드 1 - 익명 인구통계 7 (역할/학년/성별/과목/경력/기관 + opt-in 인종) → 개인식별정보 절대 없이, 자체 4글자 매칭코드로 사전·사후 페어링 - AI 사용 자신감 4 (베이스라인, 사후와 wording 동일) - 디자이너 정체성 4 (베이스라인) - 자격증명 기대 4 (사후의 가치 인식과 대칭, 미래시제) 척도는 모두 검증된 짧은 양식으로, 각 구인 ≥ 3문항 (α 추정 가능 수준). URL 파라미터로 cohort 받게 하고, UA Crimson 테마 둘 다 입혀줘. IRB 승인 전이라 활성화는 절대 하지 마.
C클코 → 당신

두 서베이를 순차 생성 — 총 60회의 API 호출 (서베이당 30회).

  • 먼저 헬퍼 모듈 + 문항 정의 파일 작성 → 재사용 가능 구조
  • 사전: 빈 셸 → 인구통계 블록 → AI-SE 베이스라인 → 디자이너 정체성 → 자격증명 기대
  • 사후: 빈 셸 → Tag → UMUX → AI-SE → 자격증명 가치 → 디자이너 정체성 → TSE 전이
  • 둘 다 cohort EmbeddedData + UA Crimson 테마 입힘
  • 둘 다 isActive=false 유지
✓이렇게 보일 거예요

두 서베이가 UA Qualtrics에 비활성으로 앉아있고, IRB 승인만 기다리는 상태.
